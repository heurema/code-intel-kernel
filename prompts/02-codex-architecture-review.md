# Codex Prompt: Architecture Review

Read the repository docs and produce `notes/architecture-review.md`.

Focus on:

1. Is the module boundary clear?
2. Are Goalrail and Punk correctly separated as consumers?
3. Is the MVP too broad?
4. Which dependencies are premature?
5. What should be implemented first?
6. What can be deleted or deferred?

Do not write implementation code in this step.
